角色主体 / SillyTavern 角色卡导入字段

这页只放角色卡主体。对应 data.name、description、personality、scenario、mes_example、creator_notes、system_prompt、post_history_instructions 等。开场白、正则、世界书不塞在这里。

开场白 / first_mes + alternate_greetings[]

酒馆里不是“只有一个开场白”：第一条导出为 first_mes，后面所有非空条目导出进 alternate_greetings[]。这里可以一直加、排序、复制。

正则 / 状态栏就在正则这里做

我按你发的酒馆截图改了正则页。上面是状态栏输出规则和示例，下面每一条正则脚本按酒馆的编辑思路来排：脚本名称、查找正则、替换为、修剪掉、作用范围、已禁用、在编辑时运行、宏替换、表层替换、最小/最大深度。

世界书 / character_book.entries[]

我也看了你后面三张世界书截图，世界书这页最好也改。所以我把条目结构往酒馆世界书编辑页靠了一些：位置、深度、顺序、触发率、关键词逻辑、扫描选项、递归选项、包含组、额外匹配来源都补进来了；角色卡 V2 标准不认识的内容，仍然会放进 extensions.sillytavern 里保留。

变量 / 自定义扩展

变量不是 Character Card V2 的主体字段，也不会自动变成酒馆聊天里的运行时变量。本页保存“变量设定/初始化宏”到 data.extensions.tavern_card_packer.variables；你可以复制本页生成的 {{.局部变量}} / {{$全局变量}} 宏放进提示词、世界书或开场白里。

导入导出 / 酒馆 PNG 与 JSON

导出 PNG 时不会生成什么信息长图，视觉保持你导入的图片。工具会把同一份 chara_card_v2 JSON 写入 PNG 元数据。导入已有酒馆 PNG 时，会尝试读取其中的 chara/Chara 数据。
